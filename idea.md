- Use nextjs for both frontend and backend
- Use supabase for database
- Use vercel for deploy 
- Admin:
  - There is default admin with default username and password
  - allow admin to change password
  - Admin can add another admin and create password for them
  - admin can add user and create password for them
- user:
  - login
  - change password
  - change avatar
- badminton center:
  - admin can CRUD center
  - center will have name, position, (will be nice if able to display map)

- booking:
  - admin can create match booking
  - user can vote join match or not 
  - admin then can update info about how many court will be and pricing

- suttlecock and drink:
  - admin later can update how many suttlecock and drink has been use for the booking

- payment:
  - will auto sumup court rent price, suttlecock price and drink price to share for all user has join 
  - User can mark they have paid or not so admin can easy track and remind them

- match & ranking:
  - User can upload the match they play
  - system will save these info and create ranking for each player, also allow to view their match history (allow both 2vs2 and 1vs1)
  - allow user to view, filter, search their match history
